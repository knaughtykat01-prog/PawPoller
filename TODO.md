## 1. Connect & verify the 4 new platforms  ⚠️ all currently "undergoing testing"

Each new platform is **built and deployed but never run against a real account** — they can't pull data until you
supply a token. Connect each under **Settings → [platform]** in the dashboard, hit **Poll Now**, and confirm posts +
counts appear. Once one works, tell Claude and it'll flip that platform from "testing" → live on the marketing site.

- [ ] **Mastodon** — on your instance: *Settings → Development → New application*, scope `read`, copy the access
      token. In PawPoller: paste **instance URL** (e.g. `https://meow.social`) + token → Connect → Poll Now.
- [ ] **Tumblr** — register an app at <https://www.tumblr.com/oauth/apps>, copy the **OAuth Consumer Key**. In
      PawPoller: enter your **blog name** (e.g. `staff` or `staff.tumblr.com`) + the key → Connect → Poll Now.
- [ ] **Pixiv** — get a one-time **refresh token** via a browser login. Easiest: `pip install gppt` then
      `gppt login` (opens a browser; copy the `refresh_token` it prints). In PawPoller: paste the refresh token
      (user ID optional) → Connect → Poll Now. Confirm illustrations **and** novels show up.
- [ ] **Threads** — the fiddly one (Meta). Create an app at <https://developers.facebook.com>, add the **Threads**
      use-case, request the `threads_basic` + `threads_manage_insights` permissions, generate a **long-lived user
      access token**. In PawPoller: paste the token (user ID optional) → Connect → Poll Now.
      - ⚠️ Meta requires app review for production access **and** restricts adult/furry content — it may come back
        empty or blocked. If it misbehaves, that's almost certainly Meta's gating, not the code.
- [ ] After verifying each: ping Claude to (a) mark it live on the marketing page, (b) drop the "in testing" badge.
